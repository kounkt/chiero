## 1. Xserverは不要になるのか

### 現状（2026-07-15 実査）
- 契約: kounkt.xsrv.jp（スタンダードプラン）／ sv7600.xserver.jp / 183.90.241.121
- ディスク使用 14.543GB / 500GB
- **9ドメインが載っている**: chiero.jp / chiero-piero.com / chieropiero.com / coaching-media.com / fx-trade-diary.com / nft-jinja.com / online-kyoiku.com / world-ai.jp / world-ai.site
- **メールアカウントは全ドメイン合計で1件**: chiero@chieropiero.com（139.68/2000MB）
- @chiero.jp のメールは**0件**（メーリングリスト0・メールマガジン0も確認済み）

### 判断
**chiero.jp だけを見れば、Xserverは不要になる。**
- サイト → GitHub Pages が肩代わり（無料）
- メール → そもそも存在しない

**しかしXserverの解約はできない。** 理由は chiero.jp ではなく、**残り8ドメインと chiero@chieropiero.com が同じ契約に載っているから**。

| Xserverを | メリット | デメリット |
|---|---|---|
| **残す**（推奨） | 8ドメインと chiero@chieropiero.com が無傷／将来 info@chiero.jp を作れる／戻す余地が残る | 年額の固定費が続く（スタンダード実勢 年13,200円前後・要確認） |
| **解約する** | 固定費ゼロ | **chiero@chieropiero.com が死ぬ**（139MBの受信履歴も）／**8ドメインの受け皿が消える**／メールを他社（Google Workspace 等・月額）へ移す作業が発生 |

→ **chiero.jp のDNSを切り替えても、Xserverの契約判断は別問題。急がなくていい。**
→ 固定費を本気で削るなら、それは「8ドメインと chieropiero.com のメールをどうするか」という別テーマ。

## 2. GitHub Pages のコストとデメリット

### コスト: 0円
Public リポジトリ + GitHub Pages は無料。今回の構成に課金要素はない。

### 上限（GitHub公式ドキュメントより・2026-07-15確認）
| 項目 | 上限 | 種類 | chiero.jpの実測 | 余裕 |
|---|---|---|---|---|
| 公開サイトのサイズ | **1 GB** | 固い制約 | **604 KB** | 1,700倍 |
| リポジトリのサイズ | 1 GB | 推奨値 | 2.1 MB（.git込み） | 500倍 |
| 帯域 | **100 GB/月** | ソフト | 1閲覧≒431KB → **月24万PVで到達** | 実測PVは29〜57 |
| ビルド | 10回/時 | ソフト | 日1回（note同期） | — |

超過時: GitHubが配信を止める場合がある／CDN導入や移設の提案連絡が来る。
→ **今の規模では、どの上限にも当たる余地がない。**

### デメリット（全部）
1. **ソースが公開される** — Public リポジトリなので誰でも中身を読める。静的サイトなので実害はないが、**秘密（APIキー・下書き・内部資料）は絶対に置けない**。※`docs/` に戦略書を置いているのは公開されている点に注意
2. **サーバー側の処理が一切できない** — PHP・フォーム送信・DB・会員機能は不可。今の設計（問い合わせはLINE）では影響なし。ただし**将来フォームを作るなら別サービスが要る**
3. **サーバー側リダイレクトが書けない** — .htaccess 相当がない。301はできず、HTMLのmetaリフレッシュ（=/shindan/でやっている方式）になる。**SEO的には301より弱い**
4. **アクセスログが取れない** — サーバーログを見られない。解析を入れるなら第三者ツール（＝プライバシーポリシーの更新が必要）
5. **GitHubの規約変更リスク** — 無料枠の条件が将来変わる可能性。ただし**正本はgitなので、Cloudflare Pages等へ数分で移設できる**（ドメインさえ持っていれば器は使い捨てられる）
6. **ビルドの遅延** — pushから反映まで数十秒〜数分
7. **障害時に手が出せない** — GitHub側の障害は待つしかない（Xserverも同様）

### メリット（全部）
1. 費用0円
2. **保守がゼロ** — PHP更新・WP更新・プラグイン更新・スパム対応が構造的に発生しない（旧WPで起きていた問題が全部消える）
3. **攻撃面が極小** — 静的HTMLのみ。乗っ取られる実行環境がない（旧WPはPHP7.4 EOL＋中リスク脆弱性1件を抱えていた）
4. **バックアップが不要** — git履歴そのものが全世代のバックアップ
5. 高速・CDN配信・HTTPS自動
6. **変更履歴が全部残る** — 誰がいつ何を変えたか追える
7. **AI全委任と相性が最良** — 「編集→push」で完結。管理画面が要らない

## 3. ドメインパワーはリセットされるか

**されない。** 理由:

- ドメインの評価（年齢・被リンク・指名検索）は**ドメイン名に紐づく**。ホスティング先の変更とは無関係
- chiero.jp は2021-11-20登録の**5年物**。この年齢は消えない
- ネームサーバーもXserverのまま（NSは触らない）

**ただし失うものはある**: 旧サイトのURL（記事7本・固定ページ24件）は404になり、そのURLが持っていた評価は消える。

**しかし実質ゼロ**: 旧サイトの実測PVは**29〜57**（Burst Statisticsで実査）。そもそも評価がほぼ無い。**失うものが無い、というのが正確**。

| | メリット | デメリット |
|---|---|---|
| ドメイン維持 | 5年の年齢・指名検索の実績・被リンクが全部残る | — |
| 旧URLの喪失 | 古い名乗り（コーチング推し）が検索結果から消える | 微小な評価を失う（実測PV 29〜57） |

## 4. 「株式会社CHIERO」で1位を取れるか

### 現状（2026-07-15 実査）
検索すると **chiero.jp は既に2位**にいる。上位は法人リスト系サイト（houjin.jp / j-bdb.com / houjin.info）。

### 1位を取るために効くこと
| 施策 | 状態 |
|---|---|
| titleに社名 | ✅ 済（`株式会社CHIERO`） |
| 構造化データ Organization | ✅ 済（法人番号・住所・創業日・sameAs） |
| **法人番号の掲載** | ✅ 済（7290001091210）— 法人リストサイトと同一法人だとGoogleに同定させる効果 |
| 登記どおりの住所 | ✅ 済（セルクル今泉404号室を補完） |
| canonical / OGP | ✅ 済 |
| **Google Search Console 登録** | ❌ **未** — DNS切替後にやる。インデックスを能動的に促せる |
| 被リンク | note・X・各アプリから張る余地あり |

### 見立て
指名検索は**競合が実質いない**（同名企業なし）。法人リストサイトは自動生成の薄いページなので、**内容のある公式サイトがインデックスされれば十分抜ける**。数日〜数週間の話。
