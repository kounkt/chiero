## 1. 結論

> **WordPressは廃止する。chiero.jp というドメインだけを残し、git管理の静的サイト（ミニマル法人サイト）を載せ替える。役割は「集客」ではなく「信頼のアンカー」。更新はAI全委任、人手は公開YES/NOとドメイン支払いの2つの檻のみ。**

- 今のWordPressは資産ではなく**負債**（保守コスト・セキュリティリスク・古い名乗りの露出）。直すのではなく退役させる
- ドメイン chiero.jp は**5年物の法人ドメイン＝唯一の本物の資産**。これは守る（期限 2026-11-30、要更新設定）
- 新サイトは既存の chiero_site リポジトリ（GitHub Pages公開済み）を土台に、Brand OS準拠のミニマル法人版へ改修して独自ドメイン接続する
- 移行後の運用コスト: サーバー費0円（GitHub Pages）・人手0分/月が定常状態

## 2. 判断基準

1. **Brand OS §1**: 法人CHIERO＝投資家の器（取得系）。名刺・契約・信頼の文脈。コーチ/コンサル語は禁止語 → 現サイトの「コンサル/コーチング」訴求と2024年のコーチング記事群は名乗りと矛盾しており、露出継続はブランド毀損
2. **本人の言葉（2026-07-15）**: 「本かSNSで知った人が、安心感や、より深く知りたいと思えることが大事」「公式サイトがちゃんと会社としてしっかりしてることは信頼や信用につながる」→ サイトのKPIはCV数ではなく**「怪しさゼロ」**
3. **OSやらないことリスト**: 薄いコンテンツ・摩耗型の毎日運用はしない → 「AIでブログ量産してSEO」という選択肢は最初から棄却
4. **全委任契約**: WordPressは構造的に摩擦の塊（PHP更新・プラグイン更新・スパム・バックアップ＝全部人手系イベント）。静的サイト＋gitはこれら全てを消せる
5. **意思決定の型**: これは畑（構造・複利）。ドメインの信頼は複利で積もり、静的化で維持コストはゼロに漸近する

## 4. 役割定義 — チエロ資産マップの中での chiero.jp

```
[声・フロー]      note / X / IG / TikTok … 日々の状態の記録（作家。AIと井戸を掘っている投資家）
                     │  「この人、会社は何なんだろう？」
                     ▼

```

- chiero.jp は**ファネルではなく身元証明**。CTAは「帽子を鳴らす」1箇所（控えめ）で十分
- 集客の主役は今後も note/X（声）と診断（器物）。chiero.jp は検索した人・本の奥付から来た人・取引先が「実在するちゃんとした会社だ」と確認して帰る場所
- 旗艦LP（v3のコーチング訴求）は**移植しない**（本人決定）。LP的要素は将来必要になったら別路地（/program/等）としてBrand OSゲート監査後に検討

### Phase 1 — 静的サイト構築と載せ替え（次セッション・AI主導）

1. **ミニマル法人サイトを構築**（chiero_site リポジトリを改修）
   - 構成（1〜3ページ）: ①トップ＝会社の三行定義＋代表プロフィール（事実系）＋事業内容（事実の列挙）＋著書・各資産リンク＋会社概要（社名・代表・所在地・設立・事業）②プライバシーポリシー ③特商法表記（既存WPから文言を引き継ぎ・監査）
   - 名乗り: Brand OS①面「投資家。経営者の意思決定の相談役」。禁止語（コーチ/コンサル/講師/指導）を全文スキャンで排除。過去実績の事実言及（「コーチングで〜の実績」）は事実系のみ可
   - デザイン: 赤 #E60012・黒・白/生成り、「楽」、赤い帽子。清潔・静か・情報が正確 ＝ 怪しさゼロの実装
   - 公開前に **brand_gate.md 4問ゲート**を通し、通過記録を残す
2. **独自ドメイン接続**（サイトだけ移す。メールのDNSは触らない）
   - リポジトリに `CNAME` ファイル（中身 `chiero.jp`）を追加、GitHub Pages設定でCustom domain＝chiero.jp・Enforce HTTPS
   - Xserver DNS設定で chiero.jp のAレコードを GitHub Pages（185.199.108.153 / 109.153 / 110.153 / 111.153）へ変更、`www` はCNAMEで `kounkt.github.io` へ。**MX・SPF（TXT）は変更しない**（メール無風）
   - 伝播確認後、https://chiero.jp が新サイトを配信していることを確認
3. **WordPress閉幕**
   - DNS切替でWPは外から到達不能になる（Xserver上には残る）。バックアップ確認後、Xserverパネルから WordPress簡単アンインストール＋DB削除（残骸を放置すると古いPHPの攻撃面が残るため消す）
   - 旧URLの跡地: 新サイト側に404を用意。アクセスがほぼゼロのためリダイレクト網羅は不要

### Phase 3 — 恒久運用と将来オプション（未来）

- **サブドメイン統合（任意・急がない）**: shindan.chiero.jp（診断）や bousai.chiero.jp（ボウサイクル）への統合は、DNSゾーンをCloudflareへ移す時にまとめて実施するのが合理的（Workersカスタムドメインの要件）。トリガー＝「Xserverメール不使用が確定し、Xserver解約を決めた時」。それまでは現URLへのリンクで十分

## 7. 反証・リスク

- **「WordPressを消すと後悔しないか」**: 全量バックアップ（.wpress＋XML）で復元可能性は残る。SEO流入は実測ほぼゼロで失うものがない。JIN:Rテーマ購入費は既にサンクコスト
- **「ミニマルすぎて機会損失では」**: 集客はサイトの仕事ではない（声と器物の仕事）。むしろ現状の「コーチング推し＋デモページ露出」の方が、経営者文脈の信頼を毎日毀損している
- **「メールが実は使われていたら」**: 本移行はMXに触れないため、確認前でもメールは壊れない。Xserver解約だけをメール確認の後段に置いた
- **「GitHub Pages依存」**: 無料サービスの仕様変更リスクはあるが、正本はgitなので載せ替え自由（Cloudflare Pages等へ数分で移設可能）。ドメインさえ守れば器は使い捨てられる
